# Triage Labels

This repo uses the default canonical triage vocabulary.

## Label mapping

- `needs-triage` -> `needs-triage`
- `needs-info` -> `needs-info`
- `ready-for-agent` -> `ready-for-agent`
- `ready-for-human` -> `ready-for-human`
- `wontfix` -> `wontfix`

## Meaning

- `needs-triage`: maintainer needs to evaluate the issue.
- `needs-info`: waiting on the reporter for more information.
- `ready-for-agent`: fully specified and ready for an AFK agent to pick up.
- `ready-for-human`: needs human implementation.
- `wontfix`: will not be actioned.

For this repo's local-markdown tracker, these labels may be represented as frontmatter values, status fields, or equivalent structured issue metadata.
